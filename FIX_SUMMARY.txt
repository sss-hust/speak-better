MICROPHONE PERMISSION FIX - EXECUTIVE SUMMARY
================================================================================

PROJECT: 会说 (Speak Better) - AI Voice Assistant
ISSUE: Microphone permission blocking Web Speech API functionality
STATUS: ✅ RESOLVED

================================================================================
WHAT WAS WRONG
================================================================================

The browser was blocking microphone access to the Web Speech API even after:
1. User clicked the voice button
2. Browser showed permission prompt
3. User selected "Allow"

The Web Speech API would still return errors instead of starting voice recognition.

ROOT CAUSE: Missing HTTP Response Header
- The server was NOT sending: Permissions-Policy: microphone=*
- Without this header, browsers deny microphone access to JavaScript code
- This is a browser security model enforcement, not a code bug

================================================================================
WHAT WAS FIXED
================================================================================

Modified: /Users/wobushi/workspace/speak-better/src/server.js

CHANGE 1 - Static File Serving (Line 128)
  Before: Headers didn't include Permissions-Policy
  After:  Added 'Permissions-Policy': 'microphone=*' to res.writeHead()

CHANGE 2 - API Response Headers (Line 158)
  Before: writeApiHeaders() didn't set Permissions-Policy
  After:  Added res.setHeader('Permissions-Policy', 'microphone=*')

================================================================================
TECHNICAL DETAILS
================================================================================

HTTP Header: Permissions-Policy: microphone=*
- Standard: W3C specification (formerly Feature-Policy)
- Browser Support: Chrome 74+, Firefox 74+, Safari 16.4+, Edge 79+
- Scope: Controls access to microphone hardware

Web APIs Affected:
1. Web Speech API (SpeechRecognition) - Primary
2. MediaDevices API (getUserMedia) - Secondary
3. WebRTC connections - Secondary

Implementation:
- Two locations in src/server.js
- Both static files AND API responses covered
- Single, minimal change: one header directive per location
